# 用户说明书：Texture Atlas 编辑器

## 环境

- 现代 Chromium / Edge / Firefox（支持 ES 模块与 Canvas）。
- 无需安装即可使用开发服务器；正式发布为静态文件，可用任意静态服务器打开。

## 启动（开发）

在 `frontend-vue` 目录执行 `npm run dev` 或使用 `dev.py`，浏览器访问终端提示的本地 URL。

## 功能概要

1. **文件 → 导入图片**：选择多张图片加入左侧列表。
2. 单击列表项查看**属性**（名称、宽高）；在列表项上**右键**可打开菜单并**删除**该项。
3. **工具栏 / 编辑菜单 → 清理全部**：清空所有已导入图片（会二次确认）。
4. **画布**：未贴图区域为**透明**；导出 PNG 亦为透明底。工具条 **「辅助线」** 下可分别勾选：**网格**、**单张上限**（紫）、**输出边界**（橙）、**图块描边**（青）；另有**线宽 (px)**（纹理像素，1～128；默认约为单张 **max(W,H) 的 1/256**，改「单张最大宽/高」会重算默认线宽；若仅再次打包且上限数字未被修正，则保留你手调的线宽）与**网格步长**。**金色**为**选中**（独立显示，不受上述四项影响）。图例随勾选项显示。标题栏含页码与单张上限；多页可用 **「全部总览」**。**单击**选中；**双击列表**将该项**图块几何中心**对齐**中间画布区**（`viewport`）的中心；必要时自动放大以便图块大致落在视区内。滚轮缩放；中键或空格+左键平移。
5. 在 **Atlas** 面板顶部设置 **单张最大宽**、**单张最大高** (px)；其下选择打包算法并 **运行打包**。超出单张范围则拆成**多页图集**。
6. **导出发布**：单页为 `atlas.json` + `atlas.png`；多页为 `atlas-00`、`atlas-01` … 成对文件（**页码从 0**，浏览器可能需允许多次下载）。
7. **逆向拆分**：选择已导出的 JSON 与 PNG，将按清单切回多张图片（每次一对文件）。

## 常见问题

- **无法导出**：请先成功运行打包且列表非空。
- **逆向失败**：确认 JSON 与 PNG 为同一导出批次且未改尺寸。
