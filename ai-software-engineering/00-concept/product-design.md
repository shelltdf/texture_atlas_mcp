# 产品设计：Texture Atlas 编辑器

## 愿景

为游戏与图形工具链提供在浏览器内编辑、打包、导出纹理图集（Texture Atlas）的桌面风格应用，并支持将已发布图集逆向拆回离散贴图。

## 目标用户

- 独立开发者与小团队技术美术
- 需要快速预览图集布局与导出 JSON/PNG 的工具使用者

## 成功标准

- 可在 Windows 风格主窗口内完成：导入图片、选择打包算法、预览画布、导出、逆向拆分；**帮助**菜单提供**支持的文件格式**说明（导入光栅图种类、导出 PNG/JSON、逆向配套要求）。
- 左侧 Dock 内图片列表带缩略图；**右键**列表可导入图片、删除、全部删除；**双击**列表项可**全图预览**原素材；选中项在列表下方展示基本属性。
- **打包图集**面板可设**单张最大宽/高**、选算法并执行打包；导入/导出图集与逆向拆分由菜单与工具栏提供。
- 画布支持**分项开关**的辅助显示（网格、上限框、输出框、图块描边）与**独立**的选中高亮，便于核对布局而不遮挡贴图。

## 图集数据等价与转换（概念）

在**信息语义**上，下列两种形态描述**同一套**精灵/贴图资源，可视为**等价**的不同载体：

- **离散像素图**：多张彼此独立的位图（及名称、顺序等语义），对应编辑器内的图片列表。
- **Atlas + 打包后的像素图（atlas + packed）**：**描述文件**（如 JSON/XML，记录布局、矩形、路径等）加上 **Pack 得到的合并位图**（一张或多张 packed 大图）。

二者之间的**可逆转换**即 **Pack** 与 **Unpack**：

- **Pack（打包）**：输入为**离散像素图**，输出为 **atlas + packed**（描述 + 合并后的像素图）。
- **Unpack（拆包）**：输入为 **atlas + packed**，输出为**离散像素图**（在应用内可表现为加入列表，或导出为多个独立文件）。

本产品的导入图集、运行打包、导出图集、逆向拆分等能力，均落在上述 Pack / Unpack 与编辑流程之上。

## 主界面布局（概念）

1. **顶部**：菜单栏、工具栏。
2. **左侧 Dock**：上部为图片缩略图列表；中部为当前选中图片属性；下部为**打包图集**（上限与算法、运行打包）；导入/导出图集与逆向在菜单与工具栏。
3. **中部**：画布区域（图集预览与布局可视化）。
4. **底部**：状态栏（就绪、选中项、画布尺寸等）。

## 范围（MVP）

- 本地文件选取导入 PNG/JPEG/WebP（浏览器支持范围内）。
- 至少两种可切换的打包算法（如简单网格、按行贪心）。
- 导出：单张图集位图 + JSON 清单（帧矩形与名称）。
- 逆向：从清单 + 图集图还原为多张下载或浏览器内 Blob。

## 非目标（当前）

- 服务器端存储、多用户协作。
- 专业游戏引擎专用格式（如 Unity `.spriteatlas`）的一键导出（可后续扩展）。
